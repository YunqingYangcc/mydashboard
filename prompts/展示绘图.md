## 仪表盘区块（信号仪表盘页面）

### 区块1: 📊 行情总览

**5列指标卡片**:
1. 产业链行情 — 多头主导/空头主导/震荡分化
2. 🟢 利好标的 — 吸筹/拉升/恐慌见底的标的数
3. 🔴 利空标的 — 派发/见顶/下跌的标的数
4. 🟡 中性标的 — 筑底/洗盘/震荡的标的数
5. 主导阶段 — 当前数量最多的阶段

**整体行情判定规则**:
- 利好 > 利空 + 中性 → 🟢 多头主导
- 利空 > 利好 + 中性 → 🔴 空头主导
- 其他 → 🟡 震荡分化

**产业链维度得分行**:
- 遍历 CHAIN_FLOW 的每个环节
- 多>空 🟢, 空>多 🔴, 否则 🟡
- 显示: 环节名 + 主导色 + 阶段 + 多空数

### 区块2: 🗺️ 产业链行情地图

**布局**: 按产业链流程横向排列，每个环节一个区块

**每个环节内**: 按市场分组（A股/美股/ETF），卡片网格展示
- 卡片: 名称 + 行情阶段色标 + 市场标签 + 量比 + 涨跌幅
- 卡片背景渐变: `linear-gradient(135deg, {phase_bg}22, rgba(255,255,255,0.01))`

### 区块3: 📋 行情阶段时序矩阵

**布局**: 表格，行=标的（按产业链分组），列=最近5个交易日

**单元格**: 每个单元格用对应阶段配色，显示 emoji + 阶段前2字
**表头**: 标的 | 日期1 | 日期2 | ... | 日期5
**标的行前缀**: 🇨🇳(A股) / 🇺🇸(美股) / 📊(ETF)

### 区块4: 🔍 标的详情（交互）

**选择器**: 下拉框，格式: `{chain} | {market} | {name}({symbol}) | {phase}`

**左列(2/5): 阶段详情面板**:
1. 大字展示: emoji + 阶段名 + label + 口诀
2. 判定依据: 量能条件 ✅/❌, 价格条件 ✅/❌
3. 量能指标: 量比(🟢>1.5x / 🟡 / 🔴<0.5x) + 价格位置(60日) + 当日涨跌
4. 操作建议: 👉 {action}
5. 投资者关注: {attention}
6. 判定理由: {reasoning}

**右列(3/5): 量价趋势图**:
1. 价格折线图(高度220px): 收盘价 + MA5 + MA20
2. 成交量折线图(高度160px): 成交量 + 20日均量
3. 放量异动标注: 量比>2.0 的日期

**数据范围**: 取最近60天绘图（数据库取120天，计算指标后取 tail(60)）

### 区块5: 📈 产业景气信号（折叠）

**综合评分展开器**: 产业景气得分 + 维度分解
**信号灯矩阵展开器**: 按维度分组，每个信号显示 emoji + 原始值 + 阈值 + 状态 + 得分

### 区块6: ⚙️ 数据管理（折叠）

**按钮**:
1. 📡 刷新所有标的行情(120天) — primary 类型
2. 🔍 重新判定所有行情阶段
3. 🔄 刷新选中标的（下拉选 + 按钮）

**指标录入表单**: metric_key + value + observed_at + name + unit + frequency + source + asset

## 组件清单

| 组件函数 | 用途 |
|---|---|
| `init_page_style()` | 全局CSS + 页面配置 |
| `metric_card(title, value, desc)` | 指标卡片 |
| `render_signature()` | 侧边栏签名(杨云清·认知升级·投资精进) |
| `render_phase_badge(phase, size)` | 行情阶段色标徽章(small/normal/large) |
| `render_phase_matrix_cell(phase)` | 矩阵单元格 |
| `render_chain_card(target, phase_data)` | 产业链标的卡片 |
| `render_chain_map(phases)` | 产业链行情地图(完整区块) |
| `render_phase_time_matrix(phases, history_phases)` | 时序矩阵(完整区块) |
| `render_volume_price_chart(df, symbol)` | 量价趋势双图 |
| `render_phase_detail(phase_data)` | 阶段详情面板 |

## 数据缓存策略

- `load_all_phases()`: `@st.cache_data(ttl=300)` — 5分钟缓存
- `load_history_phases()`: `@st.cache_data(ttl=300)` — 5分钟缓存
- 数据刷新后: `st.cache_data.clear()` + `st.rerun()`
