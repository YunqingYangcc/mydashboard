# 行情算法技能

## 触发场景

用户发出以下类型指令时激活:
- "判定行情" / "阶段判定" / "行情分析"
- "算法调整" / "修改阈值" / "改判定规则"
- "量能分析" / "行情阶段" / "重新判定"
- 任何涉及行情阶段判定逻辑的修改或执行请求

## 工作流程

### Step 1: 读取 Prompt

执行任何操作前，必须先读取 `prompts/行情算法.md`，确认:
- 9种行情阶段定义及优先级
- 量能判定阈值（is_volume_up / down / extreme）
- 中间指标计算公式
- 判定规则执行顺序
- 输出格式和存储表结构

### Step 2: 执行操作

根据用户指令执行:

| 指令 | 操作 | 代码调用 |
|---|---|---|
| 判定行情/重新判定 | 判定所有标的当前阶段 | `determine_all_current()` → `save_phases_to_db()` |
| 判定单只 | 判定指定标的 | `determine_market_phase(df)` |
| 批量历史判定 | 判定最近N天阶段 | `batch_determine_phases(symbol, df, N)` |
| 修改阈值 | 调整量能/价格阈值 | 修改代码 + 同步更新 prompt |
| 修改规则 | 调整判定优先级/条件 | 修改代码 + 同步更新 prompt |

关键代码文件: `kb/volume_analyzer.py`

### Step 3: 验证结果

判定完成后:
- 统计各阶段标的数量
- 检查是否有异常判定（如所有标的都是同一阶段）
- 展示关键标的的判定理由

### Step 4: 同步 Prompt

**核心原则**: 代码和 prompt 必须双向一致
- 修改了代码中的阈值/规则 → 必须同步更新 `prompts/行情算法.md`
- 修改了 prompt 中的规则 → 必须同步修改代码

## 9阶段优先级速查

```
1. 恐慌见底 (极端放量+暴跌+30日深跌)
2. 见顶     (量价背离)
3. 派发     (高位放量滞涨)
4. 吸筹     (低位放量未再跌)
5. 拉升     (放量上涨MA5>MA20)
6. 洗盘     (缩量回调趋势向上)
7. 筑底     (低位缩量低波动)
8. 下跌     (下跌MA5<MA20)
9. 震荡     (默认)
```
