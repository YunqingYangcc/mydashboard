## 产品概述

基于"量在价先"方法论，重构信号仪表盘为AI半导体产业链行情阶段判定系统。核心输出不是模糊评分，而是8种可枚举行情阶段（筑底/吸筹/拉升/洗盘/派发/见顶/下跌/恐慌见底），覆盖20只标的按5大产业链环节组织，给出明确操作建议和投资者关注点。

## 核心功能

1. **行情阶段判定（核心）**：每只标的独立判定当前处于8种可枚举行情阶段中的哪一种，基于量能+价格双重条件，输出明确的阶段名称+操作建议
2. **产业链行情地图**：按5大环节(GPU芯片→HBM/存储→光模块→封装→设备)横向流程展示20只标的的行情阶段，颜色编码一目了然
3. **行情阶段时序矩阵**：20只标的×最近5个交易日的行情阶段变化，观察阶段转换和产业链传导
4. **标的详情分析**：选中标的的量价趋势图+当前阶段判定依据（量能条件/价格条件各是否满足）+投资者关注清单+操作建议
5. **产业景气信号**：保留原有CapEx/TSMC营收等产业信号，作为行情判定的辅助参考
6. **8种可枚举行情阶段**：

| 阶段 | 量能特征 | 价格特征 | 操作 | 口诀 |
| --- | --- | --- | --- | --- |
| 筑底 | 低位无量，萎缩 | 横盘缓跌，新低不再 | 等待 | 低位无量就要等 |
| 吸筹 | 低位放量，明显放大 | 不再下跌甚至微涨 | 跟主力建仓 | 低位放量就要跟 |
| 拉升 | 量价齐升，持续放大 | 稳步上涨，趋势确立 | 持股加仓 | 量增价升持股 |
| 洗盘 | 缩量回调，萎缩 | 短期回调跌幅有限 | 持股不动 | 缩量上涨可持有 |
| 派发 | 高位放量滞涨 | 冲高回落或横盘 | 减仓离场 | 放量滞涨转阴 |
| 见顶 | 量价背离，价高量缩 | 高位动能衰减 | 清仓出局 | 量价背离早出局 |
| 下跌 | 放量下跌或量平价跌 | 趋势性下跌 | 空仓观望 | 量平价跌出逃 |
| 恐慌见底 | 恐慌性放量暴跌 | 长跌后加速暴跌 | 关注反转 | 恐慌抛售见底 |


默认：震荡（量能无明显特征，价格区间波动）

## 实现方案

### 一、数据库Schema扩展

在现有`knowledge_db`中新增2张表：

**1. stock_daily_quotes (日线行情表)**

```sql
CREATE TABLE IF NOT EXISTS stock_daily_quotes (
    symbol TEXT NOT NULL,
    trade_date TEXT NOT NULL,
    open REAL, high REAL, low REAL, close REAL,
    volume REAL, turnover REAL, change_pct REAL,
    market TEXT NOT NULL,        -- A股/美股/ETF
    industry_chain TEXT,         -- GPU/HBM/光模块/封装/设备/综合
    PRIMARY KEY (symbol, trade_date)
);
```

**2. market_phases (行情阶段表)**

```sql
CREATE TABLE IF NOT EXISTS market_phases (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    symbol TEXT NOT NULL,
    phase_date TEXT NOT NULL,
    phase TEXT NOT NULL,          -- 筑底/吸筹/拉升/洗盘/派发/见顶/下跌/恐慌见底/震荡
    vol_condition TEXT,           -- 量能条件判定结果
    price_condition TEXT,         -- 价格条件判定结果
    vol_ratio REAL,              -- 量比(当日量/20日均量)
    price_position REAL,         -- 价格位置(60日区间百分比)
    price_change_pct REAL,       -- 当日涨跌幅
    params_json TEXT DEFAULT '{}',-- 中间计算参数
    reasoning TEXT,              -- 判定理由
    action_suggestion TEXT,      -- 操作建议
    created_at TEXT NOT NULL,
    UNIQUE(symbol, phase_date)
);
```

**signal_definitions表扩展**：添加`industry_chain`列(用ALTER TABLE迁移)

### 二、行情阶段判定逻辑（核心算法）

8种行情阶段+1种默认的双条件判定，封装在`kb/volume_analyzer.py`中：

```python
# 核心函数签名
def determine_market_phase(df: pd.DataFrame, date: str) -> dict:
    """
    输入: DataFrame(OHLCV, 至少60日), 判定日期
    输出: {
        'phase': '吸筹',           # 行情阶段
        'vol_condition': '满足',    # 量能条件
        'price_condition': '满足',  # 价格条件
        'vol_ratio': 1.85,         # 量比
        'price_position': 0.32,     # 价格位置
        'price_change_pct': 0.8,   # 涨跌幅
        'action_suggestion': '跟随主力，逐步建仓',
        'reasoning': '低位放量(1.85x>1.5x)且价格位置32%<40%'
    }
    """
```

**判定规则优先级**（从高到低，先匹配先胜出）：

1. **恐慌见底**：vol_ratio > 2.5 AND change_pct < -5% AND 30日跌幅 > 15%
2. **见顶**：price创新高 AND vol_ratio < 0.8(前高对应量) AND price_position > 70%
3. **派发**：vol_ratio > 1.5 AND abs(change_pct) < 1% AND price_position > 80%
4. **吸筹**：vol_ratio > 1.5 AND price_position < 40% AND change_pct > -1%
5. **拉升**：vol_ratio > 1.2 AND change_pct > 1% AND MA5 > MA20
6. **洗盘**：vol_ratio < 0.7 AND -3% < change_pct < 0% AND MA5_price > MA20_price
7. **筑底**：vol_ratio < 0.5 AND price_position < 40% AND 连续3日波动<2%
8. **下跌**：change_pct < -1% AND MA5_price < MA20_price
9. **震荡**：以上均不满足

**中间指标计算**（复用，避免重复查询）：

- `vol_ratio` = 当日成交量 / MA(volume, 20)
- `price_position` = (close - low_60) / (high_60 - low_60)
- `MA5_price`, `MA20_price` = 收盘价5日/20日均线

### 三、数据获取模块

新建`kb/data_fetcher.py`，封装行情数据获取：

- `fetch_a_share_daily(symbol, start_date, end_date)` → akshare

## 实现要点

1. **行情阶段是核心输出**：不再输出-4~+4模糊评分，直接输出8+1种可枚举行情阶段之一
2. **双条件判定**：每个阶段需量能条件+价格条件同时满足，判定结果展示两个条件各是否满足
3. **优先级匹配**：8种阶段有优先级，先匹配先胜出，避免同时满足多个阶段
4. **20日均线依赖**：首次运行需获取至少60天历史数据，脚本需先填充历史
5. **数据获取频率控制**：akshare/yfinance批量获取时加sleep(0.5)间隔
6. **保留现有信号兼容**：不删除原有signal_definitions中的产业景气信号
7. **批量判定性能**：20只标的×5日 = 100次判定，复用均线中间结果避免重复查询
8. **时序展示**：最近5个交易日的阶段变化，用于观察阶段转换

## 页面布局(6个区块，自上而下)

### 区块1: 行情总览栏

5列指标卡片：产业链整体行情阶段(多数标的的阶段) + 利好标的数 + 利空标的数 + 震荡标的数 + 投资者操作建议。每列带颜色色标。

### 区块2: 产业链行情地图

横向分段流程: GPU芯片[HBM]→光模块→封装→设备，每个环节下方纵向排列龙头股卡片(分A股/美股两排)，每张卡片：股票名/代码/最新价/涨跌幅/行情阶段emoji+颜色圆点。20只标的按产业链环节分组。点击卡片跳转到区块4的详情。

### 区块3: 行情阶段时序矩阵(核心区块)

行=20只标的(按产业链分组)，列=最近5个交易日。每格显示当天的行情阶段emoji+背景色(绿/黄/红/灰)。矩阵需紧凑展示，清晰看到阶段变化和产业链传导。

### 区块4: 标的详情(交互)

左侧标的选择器(下拉框)，右侧：

- 量价趋势图：上半区价格折线+MA5/MA20，下半区成交量柱状图+20日均量线，异常日竖线标注
- 当前行情阶段大字展示 + 判定依据表(量能条件/价格条件各check/cross)
- 投资者关注清单(3-5条要点) + 操作建议

### 区块5: 产业景气信号

保留原信号灯矩阵样式，CapEx/TSMC营收等信号作为辅助参考。

### 区块6: 数据管理(折叠)

行情数据刷新按钮(一键刷新20只标的) + 手动录入 + 信号配置。
