# Analysis Validator (产业推演验证器)

## 技能概述
本技能用于检验和修正 `industry-cycle-analysis` skill 产出的推演文件的准确性。
核心功能：**搜索最新数据 → 对比验证 → 修正错误 → 更新文件**

## 触发条件
当用户：
- 要求验证推演文件的准确性
- 认为某个推演文件数据过时或错误
- 要求更新推演文件到最新数据
- 说"验证"、"检查"、"修正"、"更新"推演文件

## 工作流程

### 第一步：读取目标文件
```
读取指定的推演文件（如 data/inbox/2026-06-13/xxx推演.md）
提取文件中的关键数据点：
- 财务数据（营收、利润、毛利率等）
- 市场数据（市占率、增长率等）
- 事件数据（产品发布、融资、合作等）
- 结论和判断（周期阶段、行动建议等）
```

### 第二步：搜索最新数据
**必须搜索以下维度的最新信息：**
1. **最新财报数据**（最近1-2个季度）
   - 营收、净利润、毛利率
   - 同比/环比增长率
   - 管理层指引

2. **最新行业动态**（最近3-6个月）
   - 竞争格局变化
   - 技术突破
   - 政策法规变化

3. **最新催化剂事件**（最近1-3个月）
   - 产品发布
   - 大客户签约
   - 融资/并购
   - 监管审批

4. **最新市场情绪**
   - 分析师评级变化
   - 机构持仓变化
   - 媒体热度

### 第三步：对比验证
创建验证报告，对比原文数据和最新数据：

| 数据项 | 原文数据 | 最新数据 | 差异 | 判断 |
|--------|----------|----------|------|------|
| Q1营收 | xx亿 | xx亿 | +/-x% | ✅准确/❌错误/⚠️过时 |
| 增长率 | xx% | xx% | 差异 | 判断 |
| ... | ... | ... | ... | ... |

**验证标准：**
- ✅ **准确**：数据与最新来源一致，误差<5%
- ⚠️ **过时**：数据曾经正确，但有更新的数据（超过3个月）
- ❌ **错误**：数据与事实不符，或误差>10%
- 🔄 **需更新**：结论/判断需要根据新数据调整

### 第四步：修正并更新文件
根据验证结果，修正文件中的：
1. **错误数据** → 替换为正确数据
2. **过时数据** → 更新为最新数据
3. **过时判断** → 根据新数据重新推演
4. **添加数据来源** → 注明数据来源和时间

### 第五步：生成验证报告
输出验证摘要：

```markdown
# 验证报告：{公司名}推演.md

## 验证时间
{当前日期}

## 验证结果摘要
- 总数据点：xx个
- 准确：xx个
- 过时：xx个
- 错误：xx个
- 已修正：xx个

## 主要修正
1. {修正项1}：{原数据} → {新数据}
2. {修正项2}：{原数据} → {新数据}
...

## 结论变化
- 原结论：{原结论}
- 新结论：{新结论}
- 变化原因：{原因}

## 数据来源
- {来源1}：{时间}
- {来源2}：{时间}
...
```

## 输出规范

### 验证报告
保存路径：`data/inbox/{日期}/{公司名}验证报告.md`

### 更新后的推演文件
直接覆盖原文件，但在文件末尾添加：

```markdown
---
## 更新记录
- {日期}：验证并更新，修正xx项数据
  - 数据来源：{来源列表}
  - 主要变化：{变化摘要}
```

## 验证检查清单

### 数据准确性
- [ ] 财务数据与最新财报一致？
- [ ] 增长率计算正确？
- [ ] 市占率数据有可靠来源？
- [ ] 时间线描述准确？

### 结论合理性
- [ ] 周期阶段判断与最新数据一致？
- [ ] 行动建议与信号匹配？
- [ ] 风险识别是否完整？
- [ ] 是否遗漏重要催化剂？

### 时效性
- [ ] 数据是否超过6个月？
- [ ] 是否有更新的财报可用？
- [ ] 行业格局是否有重大变化？
- [ ] 催化剂是否已过时？

## 使用示例

**用户请求：**
```
验证 data/inbox/2026-06-13/英特尔推演.md 的准确性
```

**执行流程：**
1. 读取英特尔推演文件
2. 搜索"Intel 2026 Q1 earnings"、"Intel latest news"
3. 对比文件中的数据和最新数据
4. 修正错误/过时信息
5. 更新文件并生成验证报告

## 注意事项

1. **必须搜索最新数据**：严禁凭记忆判断，必须实际搜索
2. **保留原文逻辑**：只修正事实错误，不改变推演框架
3. **注明数据来源**：每个修正必须有来源支撑
4. **记录更新历史**：在文件末尾记录所有修改
5. **谨慎修改结论**：只有数据发生重大变化时才修改结论

## 常见问题

### Q: 数据来源冲突怎么办？
A: 优先级：官方财报 > 权威媒体 > 分析师报告 > 社交媒体

### Q: 搜索不到最新数据怎么办？
A: 在验证报告中注明"数据时效性不足"，建议用户自行补充

### Q: 结论需要大幅修改怎么办？
A: 先与用户确认，再进行修改，避免擅自改变投资建议
